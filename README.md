# Algorithmic Intraday Trading
 
The Midterm project focuses on creating a basic algorithmic trading system that generates buy, sell, or hold signals using historical stock market data.
 
The main idea behind this project is to use technical analysis indicators to analyze price movements and identify possible trading opportunities. The model uses Nifty historical data and applies indicators such as EMA (Exponential Moving Average), RSI (Relative Strength Index), and MACD (Moving Average Convergence Divergence) to generate trading signals.
 
The project also includes analysis and visualization of stock data to understand market trends and compare individual stock (Infosys) performance with the overall market.
 
The Endterm project extends this work into a full long/short algorithmic trading strategy applied to the broader NIFTY 500 index, using a different set of technical indicators - SMA (Simple Moving Average), Bollinger Bands, Z-score, Momentum, and a Stoploss for risk management - along with live market data pulled directly through the yfinance API instead of a static CSV file.
 
---
 
## Files Description
 
### 1. nifty.csv
 
This file contains historical Nifty index price data collected over the past few years.
 
The dataset is used as the primary input for the algorithmic trading model. It contains the required price information which is processed to calculate different technical indicators.
 
---
 
### 2. SOC_MidTerm.py
 
This is the main Python file containing the algorithmic trading code for the midterm project.
 
The program:
 
- Reads the Nifty data from nifty.csv
- Processes the historical price data
- Calculates technical indicators:
  - EMA (Exponential Moving Average)
  - RSI (Relative Strength Index)
  - MACD (Moving Average Convergence Divergence)
- Uses these indicators to analyze market conditions
- Generates trading signals:
  - BUY (1)
  - SELL (-1)
  - HOLD (0)
The output generated by this file is saved as final.csv.
 
---
 
### 3. midterm.csv
 
This file contains the final output generated by SOC_MidTerm.py.
 
It contains the trading signals predicted by the algorithm based on the technical indicators and price data. The signals indicate whether the strategy suggests buying, selling, or holding the stock at a particular point in time.
 
---
 
### 4. Nifty_graph.py
 
This file is used for visualization of the Nifty price data.
 
It reads the data from nifty.csv and creates graphs using Matplotlib to better understand price movement and trends over time.
 
---
 
### 5. Week0 Report.pdf
 
This report contains a stock history analysis of Infosys and its comparison with the Nifty index.
 
It includes analysis of the stock's historical performance and how it behaved in comparison with the broader market.
 
---
 
### 6. SOC_EndTerm.py
 
This is the main Python file containing the algorithmic trading code for the endterm project.
 
Unlike the midterm, which used a static CSV file and traded long-only, this program:
 
- Fetches NIFTY 500 index data (ticker ^CRSLDX) directly and live using the yfinance library, instead of reading from a saved file
- Processes and sorts the historical price data
- Calculates a different set of technical indicators:
  - SMA20 and SMA50 (Simple Moving Averages, used as trend baselines)
  - Bollinger Bands (SMA20 plus or minus 2 standard deviations)
  - Z-score (a standardized measure of how far price has stretched from SMA20)
  - Momentum (rate of change over a lookback period)
- Uses these indicators to determine market trend, strength, and overextension
- Generates a position for each day (long, short, or flat) and a result signal:
  - BUY action (1), entering a new long position, or covering an existing short
  - SELL action (-1), entering a new short position, or closing an existing long
  - HOLD (0), no action taken
- Applies a stoploss to every open position as a hard, unconditional risk control, in addition to trend and momentum based exits
The output generated by this file is saved as final_endterm.csv.
 
---
 
### 7. final_endterm.csv
 
This file contains the final output generated by SOC_EndTerm.py.
 
It contains the Date, position (1 for long, -1 for short, 0 for flat), and result (1 for buy action, -1 for sell action, 0 for hold) columns, showing the trading decisions predicted by the endterm strategy on NIFTY 500 data.

## MidTerm Project Workflow
 
1. Load historical Nifty price data.
2. Clean and process the dataset.
3. Calculate technical indicators.
4. Analyze indicator values.
5. Generate trading signals.
6. Store the results in final.csv.

## EndTerm Project Workflow
 
1. Fetch live NIFTY 500 index data using yfinance.
2. Clean and process the dataset.
3. Calculate SMA20, SMA50, Bollinger Bands, Z-score, and Momentum.
4. Determine trend direction, breakout or overextension status, and momentum strength.
5. Generate long, short, or hold decisions, applying a stoploss for risk management.
6. Store the results in final_endterm.csv.
7. Benchmark strategy performance (wealth growth, CAGR, max drawdown) against simple buy-and-hold to evaluate real-world effectiveness.
